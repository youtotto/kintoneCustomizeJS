# Documents（設計・レビュー・要件定義ドキュメント集）

kintone の設計・議論・改善を “構造化” するための  
Markdown テンプレートをまとめたディレクトリです。

- 初期ヒアリング・現状整理
- 要件定義・レビュー依頼
- 改善提案・問題提起・議論整理

など、開発プロセス全体をドキュメントで支えることを目的としています。

Toolkit の Templates タブから **Documents** を選ぶと、  
ここにある Markdown をそのままエディタで開いて編集できます。

---

## 📁 収録ドキュメント（一例）

| ファイル                              | 内容               | 主な用途           |
| --------------------------------- | ---------------- | ---------------- |
| `kintoneアプリ現状整理ドキュメント.md` | アプリ構成の棚卸・可視化     | 初期ヒアリング・引継ぎ    |
| `kintone開発レビュー依頼.md`          | 開発レビュー依頼テンプレ     | 実装後レビュー依頼      |
| `kintone要件定義書.md`               | 要件定義ドキュメントテンプレ   | 新規開発・改修要件まとめ   |
| `改善提案ドキュメント.md`               | 改善テーマの整理         | PoC・業務改善提案      |
| `問題提起ドキュメント.md`               | 問題定義と根因分析       | 問題解決の起点作り      |
| `議論整理ドキュメント.md`               | 議論の論点・合意事項の整理   | 合意形成・議事録として利用 |

※ 実際のファイル名はリポジトリの内容に合わせて調整してください。

---

## 💡 Documents を使うメリット

- Markdown で構造化されたフォーマット  
- Toolkit 上で編集 → コピペ → 共有まで完結
- AI との相性が良く、  
  - 要件定義書 → AI へのプロンプト生成  
  - 現状整理 → 改善案たたき台生成  

に使えるだけでなく、

> Templates タブ（Documents）で「AI プロンプト用テキストのダウンロード」を実行すると、  
> **このドキュメント本文に、現在開いているアプリの構成情報（Fields / Views / Graphs / Relations など）を自動で付与したテキスト** を一括で取得できます。  
> そのまま ChatGPT などの AI に貼り付ければ、  
> 「ドキュメント＋アプリ構成」の両方を前提にした要件定義・改善提案を依頼できます。

---

## 🔧 基本的な使い方

1. 目的に合った `.md` を選ぶ  
2. Toolkit またはエディタで開く  
3. `TODO:` やサンプル文言を、自分の案件内容で上書きする  
4. 必要に応じて、Templates タブから **AI プロンプト用テキストをダウンロード**し、  
   AI に投げてたたき台のドラフトを生成する  
5. クライアント・チームと共有する（kintone / Teams / Slack など）

Documents を起点にして「何を作るか」「何が問題か」を整理し、  
`js/` の JS Templates・`snippets/` の Snippets へとつなげていく、  
という使い方を想定しています。
